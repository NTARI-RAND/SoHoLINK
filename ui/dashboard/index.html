Help & Learning Center

What is SoHoLINK?

SoHoLINK is a federated compute marketplace for Small Office / Home Office (SOHO) hardware. Your node earns Bitcoin (satoshis via Lightning) by renting out spare CPU, RAM, storage, and bandwidth to verified tenants.

How does payment work?

Tenants pay per-hour using Lightning Network invoices. The node meters usage every hour and charges the tenant's account automatically. You receive 99% of the net payment (after ~2.9% processor fees); the Network Theory Applied Research Institute takes 1% as a platform fee.

How is my identity protected?

There are no passwords. Your node uses an Ed25519 DID key — a cryptographic identity. Access credentials are 84-byte signed tokens that prove identity without revealing secrets. There is no password recovery by design; if a key is compromised, revoke and re-enroll with a new DID.

What is RADIUS?

RADIUS is the protocol used by enterprise Wi-Fi (802.1X) and VPNs to authenticate users. SoHoLINK's RADIUS server (UDP 1812/1813) allows your node to act as an authentication gateway for federation tenants using your domain.

What are OPA policies?

Open Policy Agent (OPA) policies are the rules that govern what resources your node shares and with whom. They are written in Rego and embedded in the binary at build time. Edit configs/policies/resource_sharing.rego and rebuild to change them.

Where are config files?

  • C:\Program Files\SoHoLINK\config.yaml — runtime config (node name, ports, pricing)
  • Environment variables: SOHOLINK_* (override any config key)
  • Policies: embedded in binary at build time

How do I update the node?

Build a new fedaaa.exe and run scripts\install-service.ps1 as Administrator. The installer stops the old task, replaces the binary, and restarts.
